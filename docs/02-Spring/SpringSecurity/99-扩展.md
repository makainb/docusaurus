## 0. Spring Security 到底是什么？

简单来说，Spring Security 是一个 **专注于为 Java 应用提供 认证（Authentication）和 授权（Authorization）** 的安全框架。

在 Spring 生态中，它负责拦截每一个进入系统的请求，并根据你定义的规则进行身份查验和权限判定。

### 它在上述模型中扮演的角色：

1. **认证的执行者**：
它不仅支持你看到的 **Form Login**、**Basic 认证**，还内置了对接 **LDAP**、**CAS** 和 **OAuth2/OIDC** 的组件。你不需要自己写 `if(password == dbPassword)`，它帮你搞定。
2. **权限模型的守门员**：
你定义好了 **RBAC** 模型（用户有 ADMIN 角色），Spring Security 负责在你访问资源时，通过一系列“拦截器（Filter）”检查你是否有这个角色。
3. **安全防护的“防弹衣”**：
它还默认提供了对 **CSRF（跨站请求伪造）、点击劫持（Clickjacking）** 等常见 Web 攻击的防护机制，并通过安全 Header、编码规范等方式 **降低 XSS 风险** 。

---

### Spring Security 的核心工作流程（一分钟读懂）

在 Spring Security 内部，它的灵魂在于一个 **过滤器链（Filter Chain）**。

* **第一步：拦截**。当你发起请求时，它会先被过滤器截住。
* **第二步：问询（认证）**。过滤器会问：“请出示证件（用户名密码、Token、或 SSO 的票据）”。它会去查询你的 **LDAP** 或 **数据库**。
* **第三步：查表（授权）**。一旦确认你是“张三”，它会查看你的角色（**RBAC**）。如果你想访问 `/admin` 接口，它会查表发现张三只有 `USER` 角色，于是直接返回 `403 Forbidden`。
* **第四步：放行/拒绝**。规则校验通过，才让你的请求真正到达 Controller 业务层。

---

### 💡 为什么要用它，而不是自己写拦截器？

* **标准化**：它实现的是业界公认的安全标准（如 OAuth2, SAML）。
* **配置化**：通过简单的 DSL 配置（如 `http.authorizeRequests().anyRequest().authenticated()`），你就能保护整个网站，而不是给每个接口写 `if-else`。
* **深度集成**：它与 Spring Boot、Spring Cloud 无缝对接，支持方法级的权限控制（例如：只有经理能调用 `deleteAccount()` 方法）。
